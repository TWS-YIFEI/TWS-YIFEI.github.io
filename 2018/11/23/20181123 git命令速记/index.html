问题记录

  • 有时候因为本地修改的太乱了,想直接用远程库替换本地的库可以使用下面两条命令

  • 当改乱了工作区内容时,想丢弃工作区的修改 git checkout – file

  • 当改乱了文件,并且添加到了暂存区,先git reset HEAD ,再用git checkout – file
  • 当改乱了之后,并且提交到了本地版本库,使用git reset –hard 版本号 可以在各个版本之间穿梭
  • git中的check out，主要有以下两个功能：
    　　1.创建、切换分支。
    　　2.用暂存区或版本区的内容恢复到工作区。
    　　具体使用方法可查看蚂蚁部落教程。

  • 不小心提交了没用的文件到远程库

  • 每次add之前git status查看修改了哪些文件，每次只add修改过的文件，可以防止提交了无用的文件
